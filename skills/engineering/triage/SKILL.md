# Triage

将项目 issue 跟踪器中的 issue 在一套小型分类角色状态机中推进。

在分类期间发布到 issue 跟踪器的每条评论或 issue **必须**以如下声明开头：

```
> *This was generated by AI during triage.*
```

## 参考文档

- [AGENT-BRIEF.md](AGENT-BRIEF.md) — 如何撰写耐用的智能体简报
- [OUT-OF-SCOPE.md](OUT-OF-SCOPE.md) — `.out-of-scope/` 知识库的工作方式

## 角色

两个**类别**角色：

- `bug` — 某些东西坏了
- `enhancement` — 新功能或改进

五个**状态**角色：

- `needs-triage` — 维护者需要评估
- `needs-info` — 等待报告者提供更多信息
- `ready-for-agent` — 已完整定义，可交给 AFK 智能体
- `ready-for-human` — 需要人工实现
- `wontfix` — 不会处理

每个已分类的 issue 应恰好携带一个类别角色和一个状态角色。如果状态角色冲突，请标记出来并在执行任何其他操作前询问维护者。

这些是规范角色名称 — issue 跟踪器中实际使用的标签字符串可能不同。映射关系应当已经提供给你；如果尚未提供，请运行 `/setup-matt-pocock-skills`。

状态流转：未标记标签的 issue 通常首先进入 `needs-triage`；从那里可移动到 `needs-info`、`ready-for-agent`、`ready-for-human` 或 `wontfix`。报告者回复后，`needs-info` 会回到 `needs-triage`。维护者可以随时覆盖 — 如果流转看起来异常，请标记并先询问再操作。

## 调用方式

维护者调用 `/triage` 并用自然语言描述需求。解释请求并执行。示例：

- "Show me anything that needs my attention"
- "Let's look at #42"
- "Move #42 to ready-for-agent"
- "What's ready for agents to pick up?"

## 展示需要关注的内容

查询 issue 跟踪器并展示三个桶，按最旧优先：

1. **Unlabeled** — 从未分类过。
2. **`needs-triage`** — 评估进行中。
3. **`needs-info` with reporter activity since the last triage notes** — 自上次分类笔记以来报告者有活动，需要重新评估。

显示数量并为每个 issue 提供一行摘要。让维护者选择。

## 分类特定 issue

1. **收集上下文。** 完整阅读 issue（正文、评论、标签、报告者、日期）。解析任何先前的分类笔记，以免重复询问已解决的问题。使用项目的领域术语表探索代码库，并遵守该领域的 ADR。阅读 `.out-of-scope/*.md`，如果发现与本 issue 相似的先前驳回，请将其提出。

2. **给出建议。** 告诉维护者你对类别和状态的推荐及理由，并提供与 issue 相关的简短代码库摘要。等待指示。

3. **复现（仅 bug）。** 在进行任何 grill 之前，先尝试复现：阅读报告者的步骤、跟踪相关代码、运行测试或命令。报告结果 — 成功复现代码路径、复现失败，或细节不足（这是一个强烈的 `needs-info` 信号）。已确认的复现会产生更有力的智能体简报。

4. **Grill（如需要）。** 如果 issue 需要进一步充实，请运行 `/grill-with-docs` 会话。

5. **应用结果：**
   - `ready-for-agent` — 发布智能体简报评论（[AGENT-BRIEF.md](AGENT-BRIEF.md)）。
   - `ready-for-human` — 结构与智能体简报相同，但需说明为何不能委派（判断决策、外部访问、设计决策、手动测试）。
   - `needs-info` — 发布分类笔记（模板如下）。
   - `wontfix` (bug) — 礼貌解释，然后关闭。
   - `wontfix` (enhancement) — 写入 `.out-of-scope/`，从评论中链接到它，然后关闭（[OUT-OF-SCOPE.md](OUT-OF-SCOPE.md)）。
   - `needs-triage` — 应用该角色。如果有部分进展，可附加可选评论。

## 快速状态覆盖

如果维护者说 "move #42 to ready-for-agent"，请直接信任他们并应用角色。确认你将要执行的操作（角色变更、评论、关闭），然后执行。跳过 grill。如果要在未进行 grill 会话的情况下移动到 `ready-for-agent`，请询问他们是否想要撰写智能体简报。

## Needs-info 模板

```markdown
## Triage Notes

**What we've established so far:**

- point 1
- point 2

**What we still need from you (@reporter):**

- question 1
- question 2
```

将 grill 过程中已解决的所有内容记录在 "established so far" 下，以免工作丢失。问题必须具体且可执行，而不是 "please provide more info"。

## 恢复之前的会话

如果 issue 上存在先前的分类笔记，请阅读它们，检查报告者是否已回答任何未决问题，并在继续之前呈现更新后的情况。不要重复询问已解决的问题。
